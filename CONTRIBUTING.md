# コントリビューションガイド

このプロジェクトへのコントリビューションを歓迎します。
変更を行う前に、[docs/コーディング規約.md](./docs/コーディング規約.md) と [docs/設計書.md](./docs/設計書.md) を確認してください。

---

## 開発環境のセットアップ

### 必要なツール

- Node.js 18 以上
- pnpm（`npm install -g pnpm` でインストール）

### ローカルでゲームを動かす

ES Modules はファイルを直接ブラウザで開くと動作しないため、ローカルサーバーを使います。

```bash
# pnpm でサーバーを起動（npx を使う方法）
npx serve .
# または任意のローカル HTTP サーバー
```

起動後、ブラウザで `http://localhost:3000`（ポートはサーバーによる）を開いてください。

### テストの実行

```bash
pnpm test
```

`tests/` 配下のすべてのユニットテストが実行されます。

## 変更フロー

1. **Issue を作成する** — バグ報告・機能提案は先に Issue で議論する。
2. **ブランチを切る** — `main` から作業ブランチを作成する。
3. **変更を加える** — コーディング規約に従い、最小限の変更にとどめる。
4. **テストを追加・更新する** — 新しいロジックには対応するテストを追加する。
5. **テストを通す** — `pnpm test` がすべて green であることを確認する。
6. **PR を作成する** — 下記のブランチ命名・PR ガイドラインに従う。

---

## ブランチ命名規則

| 種別 | 形式 | 例 |
|------|------|----|
| 機能追加 | `feat/<説明>` | `feat/touch-controls` |
| バグ修正 | `fix/<説明>` | `fix/enemy-spawn-boundary` |
| ドキュメント | `docs/<説明>` | `docs/update-design-spec` |
| リファクタリング | `refactor/<説明>` | `refactor/extract-collision` |

---

## コミットメッセージ規則

形式: `<種別>: <変更内容の概要（日本語可）>`

| 種別 | 使いどころ |
|------|-----------|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 動作を変えないリファクタリング |
| `test` | テストの追加・修正 |
| `chore` | ビルド・設定ファイルの変更 |

例:
```
feat: タッチ操作による左右移動を追加
fix: 敵がゲームエリア外にスポーンする問題を修正
docs: コーディング規約にテスト方針を追記
```

---

## PR ガイドライン

- PR のタイトルはコミットメッセージと同じ形式で記載する。
- 変更内容・動作確認方法・スクリーンショット（UI 変更の場合）を PR 本文に記載する。
- 設計書の更新が必要な変更（新エンティティ追加、操作仕様変更など）は、設計書の更新も同一 PR に含める。
- レビュー完了後、squash merge または merge commit でマージする。

## 新機能追加のガイドライン

### ゲームバランス調整

`js/constants.js` の定数値を変更するだけで対応する。コードロジックは触らない。

### 新しいエンティティ（敵の種類・武器など）の追加

1. 必要なサイズ・速度定数を `js/constants.js` に追加する。
2. スポーン・更新・当たり判定ロジックを `js/systems.js` に追加する。
3. 描画処理を `js/render.js` に追加する。
4. CSS クラスを `style/style.css` に追加する。
5. `js/script.js` のループ内で適切な順序に呼び出しを追加する。
6. `docs/設計書.md` の該当セクションを更新する。

### 新しいゲーム画面・状態の追加

- 画面状態は `hasStarted` / `isGameOver` フラグと overlay の `hidden` クラスで管理する既存パターンに合わせる。
- ルーティングや SPA 的な画面遷移は導入しない。
