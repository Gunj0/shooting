# ゲーム設計書: Mini Shooting

## 1. 概要

ブラウザで動作するミニ縦シューティングゲーム。
プレイヤーは左右移動のみ操作し、自動発射される弾で敵を撃破してスコアを稼ぐ。

初回表示はタイトル画面とし、スペースキーでゲームを開始する。
ゲームオーバー後は、リスタートボタンまたはスペースキーで再開できる。

---

## 2. 技術制約

- HTML + CSS + Vanilla JS
- Canvas 不使用 (div 描画)
- 外部ライブラリ禁止
- 画像不使用 (色付き div で表現)
- ファイル分割構成
  - index.html
  - style/style.css
  - js/script.js
  - js/state.js
  - js/constants.js
  - js/systems.js
  - js/render.js
  - js/storage.js
  - js/utils.js

---

## 3. 画面構成

### 3.1 HUD

- SCORE: 現在スコア
- HI: 最大スコア (localStorage 永続化)
- DIFF: 現在難易度

### 3.2 ゲームエリア

- プレイヤー
- 弾
- 敵
- パワーアップアイテム
- 演出エフェクト (敵撃破 / 被弾 / パワーアップ取得)

### 3.3 オーバーレイ

- タイトルオーバーレイ
  - 初回表示時に表示
  - SPACE KEY / TAP TO START を表示
- ゲームオーバーオーバーレイ
  - FINAL SCORE 表示
  - RESTART ボタン
  - SPACE KEY / TAP: QUICK RETRY を表示

---

## 4. 操作仕様

### 4.1 キーボード

| 操作                              | キー           |
| --------------------------------- | -------------- |
| 左右移動                          | ← / → キー     |
| ゲーム開始                        | スペースキー   |
| クイック再開 (ゲームオーバー後)   | スペースキー   |
| タイトルへ戻る (ゲームオーバー後) | RESTART ボタン |

左右キーおよびスペースキーはブラウザの既定スクロールを抑止する。

### 4.2 タッチ / ポインター

| 操作                        | 動作                      |
| --------------------------- | ------------------------- |
| 画面左半分タップ / スワイプ | 左移動                    |
| 画面右半分タップ / スワイプ | 右移動                    |
| タップ                      | ゲーム開始 / クイック再開 |

---

## 5. ゲーム仕様

### 5.1 プレイヤー

- 初期位置: 画面下中央
- 画面外移動不可 (クランプ)

### 5.2 弾 (自動発射)

- 一定間隔で自動発射
- 上方向へ移動
- 画面外で削除
- 弾速は state で管理し、パワーアップ取得で上昇

### 5.3 敵

- 上部ランダム位置から出現
- 下方向へ移動
- プレイヤー接触時にゲームオーバー

### 5.4 パワーアップアイテム

- 一定時間ごとに上部から出現
- 下方向へ移動
- プレイヤーが取得すると弾速が上昇
- 弾速上昇には上限を設ける

### 5.5 当たり判定

- AABB (矩形同士) を使用
- 弾 vs 敵:
  - 敵削除
  - 弾削除
  - スコア +10
  - 敵撃破演出
- プレイヤー vs 敵:
  - ゲームオーバー
  - 被弾演出
- プレイヤー vs パワーアップ:
  - パワーアップ削除
  - 弾速上昇
  - 取得演出

### 5.6 スコア

- 敵1体撃破で +10 点
- 最大スコアは localStorage に保存

### 5.7 難易度

- 時間経過で段階的に上昇
  - 敵出現率上昇
  - 敵速度上昇
- 調整値は DIFFICULTY_SETTINGS に集約し、後から変更しやすくする

## 6. 状態設計

```ts
type Entity = {
  x: number;
  y: number;
  width: number;
  height: number;
};

type Effect = {
  x: number;
  y: number;
  size: number;
  life: number;
  maxLife: number;
  type: "enemy" | "player" | "powerup";
};

type GameState = {
  player: Entity;
  bullets: Entity[];
  enemies: Entity[];
  powerUps: Entity[];
  effects: Effect[];

  score: number;
  highScore: number;
  isGameOver: boolean;
  hasStarted: boolean;

  keys: {
    left: boolean;
    right: boolean;
  };

  lastShotTime: number;
  lastPowerUpSpawnTime: number;
  startTime: number;

  enemySpawnRate: number;
  enemySpeed: number;
  bulletSpeed: number;
  difficultyLevel: number;
};
```

- BULLET_SPEED = 7

- POWER_UP_BULLET_SPEED_BOOST
- MAX_BULLET_SPEED

DIFFICULTY_SETTINGS に以下を保持:

- spawnRateMax
- spawnRateStep
- speedStep
- speedMaxBonus
- intervalMs

## 8. メインループ

requestAnimationFrame(loop) で更新。

ゲーム進行中のフロー:

1. 難易度更新
2. 入力反映 (左右移動)
3. 弾自動発射
4. 弾の位置更新
5. 敵生成
6. パワーアップ生成
7. 敵の位置更新
8. パワーアップの位置更新
9. 当たり判定
10. エフェクト更新
11. 描画

ゲームオーバー中はゲーム進行更新を停止し、演出のみ寿命まで描画する。

---

## 9. 描画仕様

- 毎フレーム DOM 再構築 (innerHTML 初期化後 append)
- 描画対象:
  - player
  - bullets
  - enemies
  - powerUps
  - effects
- HUD は毎フレーム更新:
  - SCORE
  - HI
  - DIFF

---

## 10. ゲームオーバーと再開

- 条件: 敵とプレイヤーの接触のみ
- ゲームオーバー時:
  - isGameOver = true
  - ゲームオーバーオーバーレイ表示
  - 最大スコア更新
  - 被弾エフェクト表示
- 再開方法:
  - RESTART ボタン: タイトル画面へ戻る
  - スペースキー: 即クイック再開

---

## 11. 演出仕様

- 敵撃破時:
  - 撃破地点にリングエフェクト
- 自機被弾時:
  - プレイヤー中心に大きいリングエフェクト
  - ゲームエリアを短時間フラッシュ
- パワーアップ取得時:
  - 取得地点に専用リングエフェクト

---

## 12. 実装方針

- 可読性重視
- 関数分割を徹底
- 初学者が追いやすい直線的ロジック
- 調整値を定数へ集約し、後から変更しやすく保つ
